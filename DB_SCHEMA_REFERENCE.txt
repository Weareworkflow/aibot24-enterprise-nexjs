DATABASE SCHEMA REFERENCE - AIBOT24 (MONGODB)

========================================================================
1. COLLECTION: agents
========================================================================
Primary ID (id): {domain}-{bitrixBotId} (e.g., portal.bitrix24.es-101)
Purpose: Configuración de bots de IA individuales.

Fields:
- id: string            (Key primaria: dominio-botId)
- tenantId: string      (Dominio del portal)
- name: string          (Nombre del agente)
- type: "text"          (Tipo de interacción)
- role: string          (Rol operativo / WORK_POSITION en Bitrix)
- company: string       (Empresa / WORK_COMPANY en Bitrix)
- color: string         (Tema visual)
- systemPrompt: string  (Instrucciones específicas del agente)
- isActive: boolean     (Estado operativo)
- bitrixBotId: number   (ID numérico asignado por Bitrix24)
- avatar: string        (Opcional: Imagen en Base64)

========================================================================
2. COLLECTION: installations
========================================================================
Primary ID (id): {domain} (e.g., portal.bitrix24.es)
Purpose: Credenciales OAuth y estado del portal.

Fields:
- id: string            (Dominio del portal)
- memberId: string      (ID único de instalación de Bitrix)
- domain: string        (Dominio del portal)
- status: "active" | "suspended"
- accessToken: string   (Token de acceso actual)
- refreshToken: string  (Token de refresco OAuth)
- expiresIn: number     (Duración del token en segundos)
- expiresAt: number     (Timestamp Unix de expiración)
- clientSecret: string  (Opcional: Credencial técnica)
- clientId: string      (Opcional: Credencial técnica)

========================================================================
3. COLLECTION: config-app
========================================================================
Primary ID (id): {tenantId} (e.g., portal.bitrix24.es)
Purpose: Ajustes globales de la app por portal.

Fields:
- id: string            (Dominio del portal)
- tenantId: string      (Dominio del portal)
- theme: "light" | "dark"
- language: "es" | "en"
- systemPrompt: string  (PROMPT GLOBAL - Base para todos los agentes)

========================================================================
4. COLLECTION: metrics
========================================================================
Primary ID: Generado automáticamente por MongoDB
Relationship field (agentId): {domain}-{bitrixBotId}
Purpose: Estadísticas de uso y rendimiento por bot.

Fields:
- agentId: string       (Vínculo con AIAgent)
- usageCount: number    (Total de interacciones)
- performanceRating: number
- totalInteractionMetric: number
- latency: string       (Opcional)
- tokens: string        (Opcional)
- transfers: number     (Opcional)
- meetings: number      (Total de citas agendadas)
- abandoned: number     (Total de chats abandonados)

========================================================================
5. REST API ENDPOINTS
========================================================================

A. AGENTES
----------
- GET /api/agents?tenantId=X: Lista todos los agentes de un portal.
- GET /api/agents/[id]: Obtiene un agente (incluye prompt compilado: GLOBAL + AGENT).
- POST /api/agents: Crea o actualiza un agente (requiere id en formato dominio-botId).
- PUT /api/agents/[id]: Actualiza campos específicos de un agente.

B. CONFIGURACIÓN Y MÉTRICAS
---------------------------
- GET /api/config/[tenantId]: Obtiene la configuración global del portal.
- PUT /api/config/[tenantId]: Actualiza la configuración global.
- POST /api/metrics: Registra o actualiza estadísticas (espera agentId y metrics).
- GET /api/metrics/[agentId]: Recupera estadísticas de un bot específico.
